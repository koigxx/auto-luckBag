# Auto LuckBag

抖音直播福袋自动发现和参与工具。项目基于 Electron、Vue、TypeScript 和 Playwright，目标是在较低频率、串行访问的前提下，自动发现有福袋的直播间，记录开奖倒计时，并在临近开奖时进入直播间完成评论、粉丝团、灯牌等任务。

## 功能概览

- 自动登录复用：使用同一个 BrowserContext 复用抖音登录态。
- 自动发现直播间：从抖音直播入口页提取候选直播间，逐个低频验证。
- 可靠福袋识别：综合 WebSocket、Network Response、可见 DOM 和 ShortTouch 倒计时判断。
- 开奖倒计时：优先使用结构化响应里的 `draw_time/current_time` 计算本地开奖时间。
- 自动参与任务：
  - 自动发送评论口令；
  - 自动处理福袋弹窗里的加入粉丝团或点亮灯牌；
  - 仅在单独要求关注时点击关注，粉丝团任务不额外预先点击关注。
- 候选池：记录可靠倒计时的福袋直播间，到提前进房时间后进入参与。
- 冲突规避：开奖时间太接近的福袋不会同时加入候选池，避免同一时间只能处理一个直播间造成冲突。
- 钻石预算：所有直播间共享预算；可选“允许使用利润”，使用中奖钻石扩展预算。
- 仪表盘：展示参与数量、中奖数量、候选池、待验证数量、下次扫描时间等。
- 一键停止：停止扫描、验证、监控和页面操作，并关闭已监控页面。

## 环境要求

- Windows
- Node.js 18+
- npm

安装依赖：

```bash
npm install
```

## 启动开发模式

```bash
npm run dev
```

启动后会打开 Electron 窗口。首次使用需要点击“登录抖音”，在打开的抖音页面中完成登录。登录态会复用到后续自动扫描和直播间页面。

## 构建

```bash
npm run build
npm run build:win
```

## 使用步骤

1. 启动应用：

   ```bash
   npm run dev
   ```

2. 登录抖音：
   - 点击顶部“登录抖音”；
   - 在浏览器窗口完成登录；
   - 回到应用确认状态为“已登录”。

3. 配置基础参数：
   - 福袋类型：可选择全部、实物、钻石、其他。
   - 灯牌钻石预算：设置最多允许消耗多少钻石。
   - 允许使用利润：开启后，可消耗上限 = 钻石预算 + 已中奖钻石数量。
   - 调试日志：排查问题时开启，会打印候选按钮、点击路径等详细信息。
   - 扫描间隔：默认约 50 秒验证 1 个候选。
   - 提前进房：默认 120 秒。
   - 候选池上限：默认 5，最大 5。
   - 定时停止：可选，到时间后自动停止。

4. 点击“启动”：
   - 软件会自动扫描直播入口；
   - 发现直播间候选后逐个进入验证；
   - 确认有可靠福袋倒计时后加入候选池；
   - 到提前进房时间后进入直播间参与福袋；
   - 开奖等待结束后关闭该直播间并继续后续队列。

5. 点击“停止”：
   - 立即停止扫描、验证和直播间监控；
   - 关闭已监控房间页面；
   - 清理自动运行状态。

## 自动运行规则

自动运行是低频串行流程：

1. 扫描直播入口页，只提取直播间链接，不批量进入直播间。
2. 每轮最多验证 1 个候选直播间。
3. 验证直播间时停留约 20-32 秒，等待页面、WebSocket 和自然接口响应。
4. 只有拿到可靠倒计时的直播间才能进入主候选池。
5. 剩余时间小于等于提前进房时间时，正式进入直播间参与。
6. 验证时如果刚好临近开奖，且剩余时间位于 `(提前进房 - 20秒, 提前进房 + 15秒]`，会直接停留参与。
7. 如果开奖时间与现有候选或正在处理的直播间冲突，会跳过，避免重叠。

## 钻石预算和利润

配置项：

- 钻石预算：基础可消耗钻石数。
- 已用：当前已经用于粉丝团或灯牌的钻石数。
- 已中奖钻石：从中奖结果文本解析出的累计钻石数量。
- 允许使用利润：开启后，中奖钻石会增加可用预算。

预算规则：

```text
关闭允许使用利润：有效预算 = 钻石预算
开启允许使用利润：有效预算 = 钻石预算 + 已中奖钻石
剩余预算 = 有效预算 - 已用
```

当有效预算达到上限后：

- 需要加入粉丝团或点亮灯牌的福袋会被跳过；
- 不需要消耗钻石的福袋仍可进入候选池和参与流程。

## 福袋任务处理

评论任务：

- 优先识别福袋接口中的口令；
- 自动发送评论；
- 发送后等待评论接口或页面任务状态确认。

粉丝团或灯牌任务：

- 先检查有效钻石预算；
- 只点击福袋弹窗里的明确按钮，例如 `加入粉丝团(1钻石)`；
- 避免点击主播头像旁边的 `+加粉丝团`；
- 避免点击左侧礼物流里的 `送点亮粉丝团` 文本；
- 如果评论后福袋弹窗按钮变为加入粉丝团，会直接在当前弹窗继续，不重复点击福袋入口。

关注任务：

- 只有福袋单独要求关注且不要求粉丝团时才尝试点击关注；
- 粉丝团任务不单独预先关注，交由抖音粉丝团流程处理。

## 风控注意事项

- 默认策略偏慢，优先稳定。
- 有待验证队列时不会重复扫描入口页。
- 验证和正式进房都串行执行。
- 遇到 95152、安全验证、滑块、访问频繁等风控信号时，会暂停扫描和验证约 10 分钟。
- 不主动请求后台接口，只被动监听当前页面自然产生的 WebSocket 和 Network Response。

## 当前限制

- 抖音页面结构和接口可能变化，识别规则需要持续维护。
- 钻石中奖数量依赖中奖结果文本解析，解析不到时不会增加利润预算。
- 同一时间只处理一个正式直播间，开奖时间冲突的候选会被跳过。
- 加入粉丝团或灯牌属于真实钻石消耗操作，使用前请确认预算配置。
