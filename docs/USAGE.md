# 抖音福袋助手使用说明

## 启动

开发模式：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

Windows 打包：

```bash
npm run build:win
```

## 登录

1. 打开软件。
2. 点击“登录抖音”。
3. 在弹出的 Playwright 浏览器窗口中完成扫码或其他登录流程。
4. 登录成功后，软件会复用当前登录态。

如果登录失效，点击“退出登录”后重新登录。

## 基础配置

- 福袋类型：可选择全部、实物、钻石、其他，也可以多选组合。
- 灯牌钻石预算：所有直播间共享一个预算，预算不足时会跳过需要付费灯牌的福袋。
- 自动关注：始终启用，遇到需要关注主播的福袋会自动点击关注。
- 扫描/验证间隔：默认约 50 秒处理一个验证节奏；有待验证队列时不会重复扫描入口页。
- 提前进房：默认开奖前 25 秒进入，可配置，最小 15 秒。
- 定时停止：可选；留空表示一直运行到手动停止。

## 自动运行方式

配置完成后点击“启动”。软件会按以下流程运行：

1. 扫描直播入口页，提取直播间候选。
2. 过滤明显无效地址，例如非直播路径、保留字路径、UUID 路径。
3. 将候选放入待验证队列。
4. 每轮只进入 1 个候选直播间验证，避免快速切换触发风控。
5. 验证时会等待直播间加载，并在约 20-32 秒内多轮采样。
6. 识别依据包括 WebSocket、当前页面自然产生的福袋接口响应、可见 ShortTouch 倒计时、可见福袋入口和任务文本。
7. 软件不会主动请求额外后台接口，只监听当前页面自然产生的响应。
8. 只有拿到可靠开奖倒计时的直播间才会进入主候选池，并显示在仪表盘。
9. 如果验证通过但没有可靠倒计时，会进入低频复验队列，不会立即正式进房。
10. 候选倒计时小于等于“提前进房”配置时，正式进入直播间参与福袋。
11. 如果验证时已经临近开奖，即剩余时间小于等于“提前进房 + 15 秒”，软件会直接接管当前验证页参与，不再放回候选池。
12. 参与时会自动处理关注、评论、加入粉丝团/点亮灯牌和点击参与按钮。
13. 参与成功后会等待开奖，开奖等待结束后关闭对应直播间页面。

点击“停止”会停止全部操作，包括扫描、验证、已监控直播间、临时验证页和后台点击逻辑。

## 开奖倒计时规则

开奖倒计时是后续调度的核心依据。主候选池只接受以下倒计时来源：

- `network + exact`：来自当前页面自然产生的结构化福袋响应，例如 `/webcast/lottery/` 或包含 `lottery_info` 的响应。
- `visible-dom`：来自可见 ShortTouch 福袋入口中的倒计时，例如 `#ShortTouchLayout .ycjwPFJI` 的 `05:14`。

以下信息只作为辅助信号，不会直接驱动正式进房：

- 普通页面文本里的倒计时。
- 未知来源倒计时。
- 其他接口里的泛 `countdown` 字段。
- 只有关注、评论、粉丝团等任务文本，但没有可靠福袋倒计时。

如果只看到福袋迹象但没有可靠开奖倒计时，软件会低频复验，避免误判后反复进入无效直播间。

## 仪表盘

自动运行区域会显示：

- 参与福袋次数。
- 实物中奖次数。
- 钻石中奖次数。
- 优惠券次数。
- 中奖合计：只统计实物和钻石，优惠券不计入中奖合计。
- 候选池数量。
- 待验证数量。
- 下一次扫描/验证时间。
- 风控暂停恢复时间。
- 候选直播间名称、评分和倒计时。

## 常见日志

- `开始扫描直播入口`：正在打开直播入口页并提取候选。
- `扫描完成`：本轮发现了多少候选，并加入多少待验证。
- `待验证队列还有 ... 个候选，本轮不重复扫描入口页`：当前优先消化已有队列。
- `验证候选直播间`：正在进入一个候选直播间确认是否有真实福袋。
- `验证通过`：该直播间命中了足够的福袋信号。
- `候选未通过福袋验证`：评分或强信号不足，暂不参与。
- `识别到疑似福袋但未获得可靠开奖倒计时`：不会进入主候选池，会低频复验。
- `已记录候选`：已确认有福袋且倒计时可靠，但还没到提前进房时间。
- `验证时已临近开奖，直接在当前直播间参与`：验证过程中发现开奖很近，直接参与并等待开奖。
- `进入临近开奖直播间`：候选池中的直播间到达提前进房时间，开始正式进房参与。
- `已参与福袋，等待开奖约 ... 秒`：参与成功，当前房间进入等待开奖冷却，不会重复点击同一个福袋。
- `已安排开奖后自动关闭`：开奖等待结束后会关闭该直播间页面。
- `检测到风控信号`：发现安全验证、访问频繁等风险，自动暂停扫描和验证。
- `自动运行已停止`：用户停止或定时停止后，所有自动操作都会结束。

## 风控说明

软件不会尝试绕过安全验证、滑块或验证码。为降低触发概率，默认采用：

- 低频扫描。
- 待验证队列未清空时不重复扫描入口页。
- 单直播间串行验证。
- 只被动监听当前页面自然产生的 WebSocket 和 Network Response。
- 验证直播间之间使用导航队列控制间隔。
- 验证失败候选冷却 30 分钟。
- 缺少可靠倒计时的疑似福袋进入低频复验。
- 正式进房串行执行。
- 临近开奖时暂停新的扫描和验证。
- 遇到安全验证或访问频繁时自动冷却 10 分钟。

如果页面已经出现安全验证，需要人工处理或等待冷却，不建议频繁重启和连续点击。

## 手动添加直播间

仍然可以在“直播间”区域手动输入完整直播间 URL，例如：

```text
https://live.douyin.com/123456789
```

手动添加后，软件会进入该直播间并启动福袋监控。参与成功且拿到开奖剩余时间时，也会在开奖等待结束后自动关闭该页面。

## 常见问题

### 发现很慢

这是默认策略。软件不会快速进入大量直播间，目的是降低风控风险。可以适当增加扫描入口，但不建议把间隔调得过低。

### 明明有福袋但没识别

优先看日志里的 `signals`、`countdownSource`、`countdownConfidence` 和直播间 URL。当前策略故意收紧：没有结构化福袋响应或可见 ShortTouch 倒计时的房间不会进入主候选池。

### 候选池里有房间但还没进入

这是正常行为。软件会等待倒计时到达“提前进房”配置后再进入。倒计时未知或不可靠的候选不会直接进入主候选池。

### 进入验证页时已经快开奖

如果剩余时间小于等于“提前进房 + 15 秒”，软件会直接在当前验证页参与，并暂停后续搜索直到预计开奖后。

### 加粉丝团失败

可能是钻石预算不足、页面按钮变化、账号限制或弹窗未识别。日志会记录跳过原因。

### 优惠券算中奖吗

优惠券会单独记录，但不计入中奖合计。中奖合计只统计实物和钻石。
