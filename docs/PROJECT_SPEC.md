# 抖音福袋助手项目说明

## 目标

这是一个 Electron + Vue3 + Playwright 桌面应用，用于在用户已登录抖音后，自动发现疑似有福袋的直播间，慢速验证福袋信息，记录开奖倒计时，并在临近开奖时进入直播间完成关注、评论、加入粉丝团和参与福袋等任务。

当前默认策略是“稳妥优先”：不追求快速切换大量直播间，而是通过低频扫描、串行验证、风控冷却和临近开奖优先级，尽量降低触发抖音安全验证的概率。

## 核心流程

1. 用户登录抖音，应用保存 Playwright `storageState`。
2. 用户配置福袋类型、灯牌钻石预算、扫描间隔、提前进房时间。
3. 点击自动运行后，系统进入三阶段流水线：
   - 发现：访问直播入口页，提取直播间 URL 和可见倒计时文案。
   - 验证：每次只进入一个候选直播间，停留观察 DOM 和 WebSocket，确认是否有福袋。
   - 参与：候选倒计时进入提前进房窗口后，暂停扫描/验证，正式进入直播间参与福袋。
4. 参与成功后更新仪表盘统计；实物和钻石计入中奖，优惠券单独记录但不计入中奖合计。

## 自动发现与风控策略

- 默认扫描入口：`https://live.douyin.com`。
- 默认扫描间隔：50 秒；只在待验证队列为空时扫描入口页，队列里有候选时每轮只验证 1 个候选。
- 默认提前进房：25 秒，可配置，下限 15 秒。
- 待验证队列上限：5 个候选。
- 每轮最多验证 1 个候选直播间。
- 直播间验证导航强制串行，默认间隔 45-70 秒。
- 验证失败的候选会冷却 30 分钟，避免反复进入同一个已结束或无效直播间。
- 发现阶段会过滤 `webcast`、`live_communication`、`search` 等脚本/接口路径，避免误当成直播间。
- 正式进房也强制串行，默认间隔 4-9 秒。
- 应用启动不自动恢复上次运行时的直播间，并清空已保存房间列表，避免进入已经关闭的旧候选。
- 检测到安全验证、滑块、访问频繁、环境异常等文案时，自动运行进入 10 分钟风控冷却。
- 风控冷却期间不继续扫描或验证，但保留已确认候选，临近开奖逻辑仍按状态机优先处理。

## 福袋识别

验证阶段采用评分制，而不是单纯依赖直播间标题：

- WebSocket 命中福袋消息：强信号。
- DOM 中出现福袋入口：强信号。
- 页面出现“福袋/超级福袋/粉丝福袋/抢福袋”等文案：强信号。
- 页面出现“开奖/倒计时/等待开奖/立即参与”等文案：中等信号。
- 页面出现“关注/评论/口令/粉丝团/灯牌”等任务文案：弱信号。
- 能解析到倒计时：加分。

日志会输出 `score` 和 `signals`，便于根据真实直播间继续校准规则。

## 参与任务

正式进入直播间后由 `FudaiService` 处理：

- 自动点击福袋入口。
- 自动关注主播，始终启用。
- 如果需要评论/口令，自动发送识别到的口令；识别不到时默认发送“福袋”。
- 如果需要加入粉丝团/点亮灯牌，按全局钻石预算判断是否执行。
- 预算不足时只跳过当前需要灯牌的福袋，不停止整个自动运行。
- 点击参与后根据页面文案识别参与成功、中奖、实物、钻石和优惠券。

## 主要模块

- `src/main/auto-runner.ts`：自动运行状态机，负责发现、验证、候选池、临近开奖进房和风控冷却。
- `src/main/discovery-service.ts`：直播入口扫描和单房间福袋验证。
- `src/main/room-manager.ts`：正式直播间页面管理和监控生命周期。
- `src/main/fudai-service.ts`：福袋检测、任务处理和参与动作。
- `src/main/browser-manager.ts`：Playwright 浏览器上下文和导航节流队列。
- `src/renderer/src/components/AutoRunPanel.vue`：自动运行配置、仪表盘和候选倒计时展示。
